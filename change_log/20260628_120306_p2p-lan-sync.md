# Change Log — P2P LAN Sync

**Implements:** [`plans/20260628_102042_p2p-lan-sync.md`](../plans/20260628_102042_p2p-lan-sync.md)
**Date:** 2026-06-28

## Summary

Added opt-in, user-initiated, same-LAN, device-to-device account+group transfer (P2P sync),
ported from the SMS Sentry Kotlin guide and adapted to this Flutter app. Security is at the payload
layer: a per-session ~320-bit pairing code (shown on the host, typed into the client out-of-band,
never on the wire) is stretched with PBKDF2-HMAC-SHA256 (300k iterations) into an AES-256-GCM key.
Phase 1 is one-directional (host pushes → client imports). Two refinements from review: the host
binds a **random OS-assigned port** (displayed to the user), and hosting **auto-stops** after a
configurable idle timeout.

## New files

- `lib/services/p2p_sync_service.dart` — crypto (pairing code gen/normalize/format, PBKDF2 derive,
  AES-GCM wire encrypt/decrypt), bounded socket line reader, host server + client logic, and
  payload validation caps. Reuses `encrypt` + `pointycastle` already in the project. Never logs
  secrets or payloads.
- `lib/providers/sync_provider.dart` — `SyncState` model (Idle/Hosting/Connecting/Syncing/
  Completed/Error) and orchestration; builds the host payload (device-key decrypt → JSON) and routes
  received data to the import funnel.
- `lib/screens/sync_screen.dart` — host/join UI: host shows IP + random port + grouped pairing code
  (copy + idle-timeout note); join takes IP + port + code. Lock-aware auto-pop; tears down the
  listener on dispose.
- `lib/utils/network_utils.dart` — `getLocalIpAddress()` (first non-loopback IPv4).
- `test/services/p2p_sync_service_test.dart` — pairing-code entropy/alphabet, normalize/format,
  wire crypto round-trip, wrong-code rejection, validation caps, and a loopback host↔client
  round-trip (happy path + wrong-code).

## Security notes

- OTP secrets cross the wire only as AES-256-GCM ciphertext keyed by the out-of-band code; a wrong
  code fails GCM tag verification (auth = decrypt), with no fallback/downgrade.
- Received secrets are re-encrypted under the receiving device's own key by
  `AccountsProvider.importData` before storage; plaintext is transient and never logged.
- Hostile-peer hardening: bounded line reads (4 KB handshake / 16 MB payload), socket/connect
  timeouts, and pre-ingestion payload caps.
- Random port is conflict avoidance / mild defense-in-depth only — documented as **not** a security
  boundary; security rests on the pairing code and GCM sealing.
